# VRM 系统架构升级 — 最终方案总览

> 本文档是 ATRI Chat 项目 VRM 系统架构重设计的最终定稿。
> 整合了 7 轮讨论（Round 0 ~ Round 6）的所有有效内容，消除了因上下文遗忘造成的重复与遗漏。

---

## 一、现有架构核心问题

### 1.1 单体 Hook 综合症
`useVRM.ts`（707行）同时承担模型 URL 管理、动作绑定 API、音频队列播放、字幕显示、表情切换、闲置计时器调度、动作分类查找等所有职责。任何一个功能的改动都可能引发连锁反应。

### 1.2 命令式编排的脆弱性
`playNextSegment` 函数是一个递归的、依赖 Promise 链的编排器，测试困难、中断脆弱、扩展困难。

### 1.3 Store 与逻辑反向耦合
`useVRM`（逻辑 Hook）既**读**状态又**写**状态，同时暴露回调给子组件，形成三角耦合，状态流向不唯一。

### 1.4 通信层手动维护
后端手写 SSE 格式 → 前端手动解析 `astream_events`，与 LangGraph 标准协议重复，难以扩展工具调用 UI、时间旅行等能力。

### 1.5 VRM 模式非流式
VRM 模式使用 `agent.ainvoke`（非流式）获取全文后再 TTS，无法实现文本流式输出。

---

## 二、设计哲学

> **"状态只描述现实，命令负责改变，指挥者负责编排"**

引入三层分离架构：**对话层 → 指挥层 → 渲染层**

|---|--------|----------|----------|
| 1 | Director 实例化方式 | **Class 单例，存储在 Zustand Store 中** | R2/R3 |
| 2 | R3F 层通信方案 | **混合架构** — Store 桥接状态，Director 暴露 audioElement getter | R2 |
| 3 | 有序/并行命令执行 | 叙事音频有序队列；表情/镜头/SFX 并行旁路 | R2/R3 |

| 5 | AI 指令系统 | **双通道**：工具调用通道（立即） + 文本内嵌标记通道（有序） | R3 |
| 6 | AudioElement 归属 | Director 持有，通过 getter 暴露，R3F 层通过 hook 获取 | R3 |
| 7 | motionFinished 事件传递 | 写 Store 事件字段（时间戳），Director 订阅 | R3 |
| 8 | 通信协议 | **`@langchain/langgraph-sdk` 的 `useStream`** 替代自定义 SSE | R4 |
| 9 | 后端框架 | 保留 FastAPI + **混合兼容 useStream 协议** | R4/R5 |
| 10 | 双向状态同步 | 前端 submit 时自动注入 VRM 场景上下文；后端中间件注入 system message | R4 |
| 11 | TTS 处理方式 | TTS 解耦为后处理 Hook，通过 `onCustomEvent` 推送音频段 | R5 |
| 12 | 标记语法升级 | 从 `[State:xxx]` 升级为 `<emo/>` `<act/>` `<pau/>` XML 标签，兼容旧格式 | R3 |
| 13 | 场景配置系统 | 新增 `VRMSceneConfig` / `VRMCharacterConfig` / `VRMCameraConfig` | R6 |
| 14 | 迁移策略 | **一次性重写**（新分支） | R3 |
| 15 | 多实体管理 | **Entity Registry** — Store 中维护 `entities: Record<id, EntityState>` 字典，支持 VRM 角色 + GLTF 道具 + 静态模型共存 | R7 |
| 16 | 移动系统 | **复合原子操作** — Director 发出 MOVE 命令 → MotionBridge 自动播放行走动画 + R3F useFrame 程序化位移 | R7 |
| 17 | 资源管理 | 模型通过 URL 驱动 React 生命周期加载/卸载；动作使用 LRU 缓存池；背景列表从后端 API 获取 | R7 |
| 18 | 状态归属原则 | AI 需感知或需持久化的 → 全局 Store；纯渲染细节（blend权重/加载进度等） → 组件本地 Ref/State | R7 |
| 19 | 预览模式 | 提供轻量 `VRMPreviewer` 组件，复用渲染层但**不依赖 Director/通信层**，通过 props 直接驱动 | R7 |
| 20 | 桌宠模式 | 基于 **Tauri 多窗口机制**：主窗口跑逻辑，完全透明子窗口跑轻量级 `VRMPreviewer`，使用 Tauri IPC 同步状态 | R8 |

## 四、新架构全景图

```
┌──────────────────────────────────────────────────────────────────┐
│                 Layer 1: 对话层（Conversation）                   │
│                                                                  │
│  useAgentStream (@langchain/langgraph-sdk → useStream)           │
│  ├── stream.messages          // 消息历史（AI/Human/Tool）       │
│  ├── stream.toolCalls         // 工具调用生命周期                │
│  ├── stream.submit(input)     // 发送消息 + VRM 场景上下文      │
│  ├── stream.stop()            // 中断当前生成                   │
│  └── onCustomEvent            // 接收 VRM 音频段                │
│                                                                  │
│  StreamAdapter（对话层 ↔ 指挥层的桥梁）                          │
│  ├── 监听 toolCalls → 过滤 perform_actions → dispatch Director  │
│  ├── 监听 onCustomEvent → 解析 performance tags → dispatch      │
│  └── submit 时自动注入 VRMWorldStore 场景快照                   │
│                                                                  │
└──────────────────────┬───────────────────────────────────────────┘
                       │ dispatch(VRMCommand)
┌──────────────────────▼───────────────────────────────────────────┐
│                 Layer 2: 指挥层（Director）                       │
│                                                                  │
│  VRMDirector (Class 单例, 存于 Zustand)                          │
│  ├── CommandRouter（按类型/优先级分发）                          │
│  ├── EngagementStateMachine (XState)                             │
│  └── Controllers                                                │
│      ├── NarrativeController（叙事队列 + 音频管理）             │
│      ├── ExpressionController（情感表情 — 仅情感层）             │
│      ├── MotionBridge（动作指令 → Store → R3F）                 │
│      ├── CameraController（镜头预设与过渡）                     │
│      ├── GazeController（注视目标控制）                         │
│      ├── SceneController（背景/地面配置）                       │
│      └── SFXController（音效播放）                              │
│                                                                  │
└──────────────────────┬───────────────────────────────────────────┘
                       │ 写 Store
┌──────────────────────▼───────────────────────────────────────────┐
│                 Layer 3: 渲染层（R3F + UI）                      │
│                                                                  │
│  VRMWorldStore（Zustand）— 全局单一状态源                        │
│  ├── entities: Record<id, EntityState>  // 多实体注册表          │
│  ├── engagement / emotion / motion / camera / subtitle          │
│  ├── scene / character / gaze / posture                          │
│  ├── renderConfig / sceneConfig / characterConfig / cameraConfig │
│  └── _events（motionFinished 等回写通道）                       │
│                                                                  │
│  R3F 渲染组件（只读 Store、内含自治程序）                        │
│  ├── AIStage.tsx → EntityFactory（按 entities 字典动态渲染）    │
│  │   ├── Character.tsx (type=vrm) → 完整VRM逻辑                │
│  │   └── Prop.tsx (type=prop) → 轻量GLTF（基础动画、无表情）   │
│  ├── useMotionController / useAutoBlink / useLipSync            │
│  ├── useAutoLookAt（读 Store.gaze.target）                      │
│  └── VRMPreviewer.tsx — 轻量预览（无Director，props直驱）       │
│                                                                  │
│  UI 组件                                                         │
│  ├── 字幕面板                                                    │
│  ├── VRMRenderSettings（Tab: 光照/场景/角色/相机/高级）         │
│  └── 状态指示器                                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 五、执行阶段概览

| 阶段 | 名称 | 核心目标 |
|---|---|---|
| **阶段 0** | 场景配置基础 | 新增 Scene/Character/Camera Config 类型，扩展 Store 与 Settings UI（独立于通信层，可先行） |
| **阶段 1** | 通信层替换 | `useStream` 替代自定义 SSE，跑通基础对话 |
| **阶段 2** | VRM 指挥层 | Director + Controllers + XState 状态机，跑通 VRM 表演 |
| **阶段 3** | VRM 音频 + 双向同步 | TTS → onCustomEvent 推送，前端完整播放，场景上下文注入后端 |
| **阶段 4+** | 增强功能 | 时间旅行 UI、消息分支、Human-in-the-loop、物体交互等 |
