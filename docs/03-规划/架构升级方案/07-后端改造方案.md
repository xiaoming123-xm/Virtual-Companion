# 后端改造方案

---

## 一、当前后端架构分析

### 1.1 核心链路

```
api/routes/messages.py → POST /messages
  → AgentCoordinator.send_message()
    → OutputStrategy 策略分发:
      ├─ Text模式: TextOutputStrategy → MessageService.run_streaming → astream_events
      └─ VRM模式:  VRMOutputStrategy → agent.ainvoke (非流式) → VRMService.generate_stream
```

### 1.2 关键发现

| 模块 | 当前职责 | 问题 |
|---|---|---|
| `AgentCoordinator` | 统一消息入口，管消息保存、标题更新、SSE累积 | SSE累积逻辑和消息保存耦合 |
| `OutputStrategy` | 分 Text/VRM 两种策略 | VRM策略是**非流式的**（先全量获取再TTS） |
| `VRMService` | 按换行分句 → 去标记 → TTS → base64 | 职责清晰但和OutputStrategy绑定 |
| `MessageService` | `astream_events` → 转前端 payload | 手动解析事件，与 useStream 重复 |
| `messages.py` (API) | FastAPI StreamingResponse (SSE) | 手写 SSE 格式 |

### 1.3 关键中间件（保留不动）

- `select_model_and_params` — 动态切换模型 ✅
- `build_character_prompt` — 动态角色提示词 ✅
- `filter_tools_by_mode` — 按 VRM/Text 过滤工具 ✅

---

## 二、改造原则

1. **`createAgent` + 中间件保留不变** — 已经是好的架构
2. **统一为流式输出** — VRM模式也流式，不再先全量获取
3. **TTS 解耦为独立后处理** — 通过 `onCustomEvent` 推送
4. **消息保存交给 LangGraph checkpointer** — 不再手动保存
5. **兼容 `useStream` 协议** — Agent API 遵循 LangGraph API 标准

---

## 三、改造详情

### 3.1 消除 MessageService 和手写 SSE

**之前：** `MessageService.run_streaming()` 手动解析 `astream_events` → 转 JSON → 手写 SSE
**之后：** `useStream` 直接连接兼容端点，自动处理消息流

### 3.2 统一 OutputStrategy → TTS 后处理 Hook

**之前：** VRM模式用 `agent.ainvoke` 非流式获取全文 → TTS
**之后：** 所有模式统一流式 → Agent 完成后触发 TTS 后处理

```python
class TTSPostProcessor:
    """AI 回复完成后的 TTS 后处理器

    通过 LangGraph 的 writer (custom events) 将音频段推送到前端
    """

    def __init__(self, vrm_service: VRMService):
        self.vrm_service = vrm_service

    async def process_response(
        self,
        ai_response: str,
        character_id: str,
        db_session: Session,
        writer,  # LangGraph 的 custom event writer
    ):
        """在 Agent 回复完成后，将文本送去 TTS 并通过 custom event 推送"""
        async for segment in self.vrm_service.generate_stream(
            ai_response, character_id, db_session
        ):
            writer.write({
                "type": "vrm_segment",
                "data": segment
            })

        writer.write({
            "type": "vrm_complete",
            "total_segments": segment["sentence_index"] + 1
        })
```

### 3.3 VRMService 标记解析升级

`VRMService` 当前设计已经比较好（分句 → 去标记 → TTS → 流式 yield），唯一改动是支持新 XML 标签格式：

```python
class VRMService:
    # 旧标记
    LEGACY_MARKUP = re.compile(r'\[[^\]]+:[^\]]+\]')
    # 新标记
    XML_MARKUP = re.compile(r'<[a-z]+\s+[^/]*/>')

    def strip_markups(self, text: str) -> str:
        """移除所有标记，保留纯文本（用于 TTS 合成）"""
        text = self.LEGACY_MARKUP.sub('', text)
        text = self.XML_MARKUP.sub('', text)
        return text.strip()
```

### 3.4 混合方案：FastAPI + useStream 兼容

保留 FastAPI 路由，但让其输出 `useStream` 期望的协议格式：

```python
# api/routes/agent_stream.py — 新路由

from fastapi import APIRouter, Request
from fastapi.responses import StreamingResponse

router = APIRouter()

@router.post("/agent/stream")
async def agent_stream(request: Request, ...):
    """useStream 兼容的流式端点

    useStream 期望的 SSE 格式:
    - data: {"event": "values", "data": {...state...}}
    - data: {"event": "updates", "data": {...}}
    - data: {"event": "messages/partial", "data": [...]}
    - data: {"event": "custom", "data": {...}}  ← VRM 段通过这里推送
    """
    async def generate():
        stream = await agent.stream(
            {"messages": [{"role": "user", "content": content}]},
            config=config,
            context=context,
            encoding="text/event-stream",
            stream_mode=["values", "updates", "messages", "custom"],
        )
        async for chunk in stream:
            yield chunk

    return StreamingResponse(generate(), media_type="text/event-stream")
```

### 3.5 VRM 上下文注入中间件

```python
# core/middleware/vrm_context.py — 新中间件

@wrap_model_call
async def inject_vrm_context(request, handler):
    """将前端提交的 VRM 上下文注入到对话中"""
    context = request.runtime.context
    vrm_state = context.vrm_context

    if vrm_state:
        scene_msg = f"""[当前场景状态]
- 自身: 状态={vrm_state.get('engagement')}, 情感={vrm_state.get('emotion')}
- 注视: {vrm_state.get('gaze')}
- 附近物体: {vrm_state.get('nearby_objects', [])}
"""
        msgs = list(request.messages)
        msgs.insert(-1, {"role": "system", "content": scene_msg})
        return await handler(request.override(messages=msgs))

    return await handler(request)
```

### 3.6 LangGraph State 扩展

from typing import TypedDict, Optional, Annotated
