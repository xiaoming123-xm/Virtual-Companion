## 四、风险与注意事项

### 4.1 useStream 连接方式

`useStream` 默认连接 `http://localhost:2024` (LangGraph Server)。
如果不用 `langgraph dev`，需要在 FastAPI 中实现兼容的 streaming 协议。

**建议先用 `langgraph dev` 跑通，再考虑是否需要嵌入 FastAPI。**

### 4.2 消息保存

LangGraph checkpointer 已经保存了完整的 state（包括 messages）。
但如果业务需要写自己的 `conversations`/`messages` 表，需要在 `onFinish` 回调中同步。

### 4.3 TTS 延迟

从流式文本到 TTS 再到前端有延迟。需要在 UI 上给出 "正在生成语音..." 的状态。

### 4.4 标记兼容

阶段初期同时支持 `[State:xxx]` 和 `<emo/>` 两种格式，后期逐步迁移。

### 4.5 XState 学习成本

XState v5 API 与 v4 差异较大。建议在实施前先熟悉 v5 的 `createMachine` + `createActor` 模式。

### 4.6 测试策略

- Director + Controllers 是纯 Class，可以直接单元测试
- 状态机可以用 XState 的 `testModel` 做模型测试
- R3F 层变动较小（只改了数据源），可以用集成测试覆盖
