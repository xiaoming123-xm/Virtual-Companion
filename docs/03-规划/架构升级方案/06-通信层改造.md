# 通信层改造（useStream 集成）

---

## 一、改造核心：用 `useStream` 替代自定义 SSE

### 1.1 当前问题

| 当前 | 问题 |
|---|---|
| `MessageService.run_streaming()` 手动解析 `astream_events` | 与 LangGraph 标准协议重复 |
| `messages.ts` 手写 SSE 解析逻辑 | 脆弱、难维护 |
| 前端只支持单向（后端→前端） | 无法注入场景上下文 |
| 工具调用渲染需手动 `onToolStart/onToolEnd` | 无生命周期管理 |
| 无时间旅行/消息分支能力 | |

### 1.2 改造后

| 改造后 | 优势 |
|---|---|
| `useStream` 自动处理 messages/toolCalls/reasoning | 无需手动解析 |
| `onCustomEvent` 接收 VRM 音频段 | 标准扩展方式 |
| `submit(input)` 可携带场景上下文 | 双向同步 |
| `toolCalls` 带生命周期（pending/completed/error） | UI 卡片自动映射 |
| 可启用 `fetchStateHistory` | 时间旅行支持 |

## 二、SSE 协议变更

### 2.1 当前 SSE 类型

```
type: 'text'          // 纯文本流
type: 'reasoning'     // 推理过程
type: 'status'        // 状态消息
type: 'tool_start'    // 工具开始执行
type: 'tool_result'   // 工具执行结果
type: 'vrm_segment'   // VRM 音频段
type: 'vrm_complete'  // VRM 播放完毕
```

### 2.2 新协议（useStream 标准）

```
useStream 自动处理（无需手动解析）：
- event: "values"              → state 快照
- event: "updates"             → state 增量更新
- event: "messages/partial"    → AI 消息流式 token
- event: "messages/complete"   → AI 消息完成

VRM 扩展（通过 custom event）：
- event: "custom"  →  { type: "vrm_segment", data: { ... } }
- event: "custom"  →  { type: "vrm_complete", total_segments: N }
```

### 2.3 VRM Custom Event 数据结构

```typescript
// 音频段事件
interface VRMSegmentEvent {
  type: "vrm_segment";
  data: {
    sentence_index: number;
    marked_text: string;       // 含 <emo/><act/><pau/> 标签
    audio_url: string | null;
    duration_ms: number;       // 音频时长（新增，用于预排）
  };
}

// 播放完毕事件
interface VRMCompleteEvent {
  type: "vrm_complete";
  total_segments: number;
}

type VRMCustomEvent = VRMSegmentEvent | VRMCompleteEvent;
```

## 三、双向状态同步

### 3.1 为什么需要双向同步

**当前痛点：** 每次 AI 想知道"角色在哪、面朝哪、附近有什么"，都需要调用 `GetGlobalSceneContext` 工具。
- AI 的第一反应永远是"我不知道环境"→ 先工具调用 → 再决策
- 每轮对话多一次工具调用 = 多一次 LLM 推理 = 更慢、更贵
- 环境信息不连续

### 3.2 解决方案：Context Injection

前端每次 submit 时，自动附带当前场景快照：

```typescript
stream.submit({
  messages: [{ type: "human", content: userInput }],
  vrm_context: {
    character: {
      engagement: "idle_initial",
      emotion: "neutral",
      posture: "standing",
      gaze: "at_user",
      position: { x: 0, y: 0, z: 5 },
    },
    scene: {
      nearby_objects: [
        { id: "cup_blue", name: "蓝色杯子", dist: 1.2 },
      ],
    },
    camera: { mode: "front", dist: "mid" },
  }
});
```

AI 收到消息时**立刻拥有完整的环境感知**，无需工具调用。

### 3.3 后端注入流程

```python
# core/middleware/vrm_context.py

async def inject_vrm_context(request, handler):
    vrm_state = request.runtime.context.vrm_context

    if vrm_state:
        scene_msg = f"""[当前场景状态]
- 自身: 状态={vrm_state.get('engagement')}, 情感={vrm_state.get('emotion')}
- 注视: {vrm_state.get('gaze')}
- 附近物体: {vrm_state.get('nearby_objects', [])}
"""
        msgs = list(request.messages)
        msgs.insert(-1, {"role": "system", "content": scene_msg})
        return await handler(request.override(messages=msgs))

    return await handler(request)
```

## 四、工具调用 UI 卡片

利用 `useStream` 的 `ToolCallWithResult` 模式，为 `perform_actions` 构建专属 UI 卡片：

```typescript
// components/chat/ToolCard.tsx

function ToolCard({ toolCall }: { toolCall: ToolCallWithResult }) {
  if (toolCall.state === "pending") {
    return <LoadingCard name={toolCall.call.name} args={toolCall.call.args} />;
  }

  if (toolCall.state === "error") {
    return <ErrorCard name={toolCall.call.name} error={toolCall.result} />;
  }

  switch (toolCall.call.name) {
    case "perform_actions":
      return <PerformActionsCard actions={toolCall.call.args.actions} />;
    case "memory_save":
      return <MemorySaveCard result={toolCall.result} />;
    case "memory_search":
      return <MemorySearchCard args={toolCall.call.args} result={toolCall.result} />;
    default:
      return <GenericToolCard toolCall={toolCall} />;
  }
}

// perform_actions 的专属渲染
function PerformActionsCard({ actions }: { actions: ActionItem[] }) {
  return (
    <div className="action-sequence-card">
      <div className="card-header">🎬 角色动作</div>
      <div className="action-timeline">
        {actions.map((action, i) => (
          <div key={i} className="action-step">
            <ActionIcon type={action.type} />
            <span>{getActionLabel(action)}</span>
          </div>
        ))}
      </div>
    </div>
  );
}
```

## 五、完整数据流（从用户输入到角色表演）

```
用户输入 "帮我倒杯水"
    │
    ├─→ useAgentStream.submitWithContext("帮我倒杯水")
    │     → stream.submit({
    │         messages: [{ type: "human", content: "帮我倒杯水" }],
    │         vrm_context: { engagement: "idle_initial", nearby_objects: [...] }
    │       })
    │
    ├─→ director.dispatch({ type: 'ENTER_THINKING' })
    │       → XState: IDLE → THINKING
    │       → 播放思考动作
    │
    │                    ┌─── LangGraph Agent 处理 ───┐
    │                    │  看到 vrm_context：         │
    │                    │  "附近有 cup_blue (1.2m)"   │
    │                    │  无需工具调用，直接规划！    │
    │                    │                             │
    │                    │  Step 1: Tool Call           │
    │                    │    perform_actions({         │
    │                    │      actions: [              │
    │                    │        { type: "gaze",       │
    │                    │          target: "cup_blue"} │
    │                    │        { type: "camera",     │
    │                    │          mode: "ots" }       │
    │                    │      ]                       │
    │                    │    })                        │
    │                    │                             │
    │                    │  Step 2: Text Stream         │
    │                    │    "<emo v='happy'/>好的！   │
    │                    │     我去拿<act v='walk'/>。" │
    │                    └─────────────────────────────┘
    │
    ├─→ useStream.toolCalls 更新：
    │   → StreamAdapter.handleToolCalls()
    │   → director.dispatch(SET_GAZE, CAMERA_CUT)  ← 立即执行
    │   → UI 渲染 PerformActionsCard
    │
    ├─→ onCustomEvent: vrm_segment
    │   → StreamAdapter.handleVRMSegment()
    │   → 解析 <emo v='happy'/> → director.dispatch(SET_EMOTION)
    │   → 解析 <act v='walk'/> → director.dispatch(PLAY_MOTION)
    │   → 纯文本 → director.dispatch(NARRATE_SEGMENT)
    │   → XState: THINKING → NARRATING
    │   → NarrativeController: 播放音频 + 字幕
    │
    ├─→ 叙事完成
    │   → XState: NARRATING → IDLE_INITIAL
    │   → 重新播放初始动作 + 启动闲置计时器
    │
    └─→ stream 结束
```

## 六、时间旅行与 VRM 的结合（后续增强）

因为在 `AgentState` 中加入了 `vrm_context`，LangGraph checkpoint 回溯时 VRM 世界状态也能恢复：

```typescript
function handleCheckpointSelect(checkpoint: ThreadState) {
  // 1. 恢复对话状态（messages）
  stream.submit(null, { checkpoint: checkpoint.checkpoint });

  // 2. 恢复 VRM 世界状态
  const vrmState = checkpoint.values?.vrm_context as VRMContext;
  if (vrmState) {
    director.dispatch({ type: 'SET_EMOTION', emotion: vrmState.emotion });
    director.dispatch({ type: 'SET_GAZE', target: vrmState.gaze_target });
  }
}
```
