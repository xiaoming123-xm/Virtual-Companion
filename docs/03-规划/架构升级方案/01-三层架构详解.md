# 三层架构详解

---

## 一、架构总览

新架构采用严格的三层分离：

```
对话层 (Conversation Layer)
    ↓ dispatch(VRMCommand)
指挥层 (Director Layer)
    ↓ 写 Store
渲染层 (Rendering Layer)
```

**核心原则：**
- 层间通信**只通过命令和 Store**，不存在跨层直接调用
- **对话层**只关心"什么时候发什么命令"
- **指挥层**只关心"命令如何编排和执行"
- **渲染层**只关心"读 Store 状态渲染画面"

---

## 二、Layer 1: 对话层（Conversation Layer）

### 2.1 职责
- 管理用户与 AI 的对话流（消息、工具调用）
- 将 AI 的输出（文本、工具调用、TTS音频）转换为 VRMCommand
- 在发送消息时自动注入 VRM 场景上下文

### 2.2 核心组件

#### `useAgentStream` Hook
封装 `@langchain/langgraph-sdk` 的 `useStream`，作为 `useChat.ts` 的替代品。

```typescript
// hooks/useAgentStream.ts

import { useStream } from "@langchain/langgraph-sdk/react";

interface AgentState {
  messages: BaseMessage[];
}

export function useAgentStream(options: {
  conversationId: string;
  characterId: string;
  onVRMSegment?: (segment: VRMSegmentData) => void;
  onVRMComplete?: () => void;
}) {
  const director = useVRMWorldStore(s => s.director);

  const stream = useStream<AgentState>({
    apiUrl: API_URL,
    assistantId: "atri_companion",
    threadId: options.conversationId,

    // VRM 音频段通过 custom event 接收
    onCustomEvent: (event: unknown) => {
      const evt = event as VRMCustomEvent;

      if (evt.type === "vrm_segment") {
        const { text, commands } = parsePerformanceTags(evt.data.marked_text);
        commands.forEach(cmd => director?.dispatch(cmd));
        director?.dispatch({
          type: 'NARRATE_SEGMENT',
          payload: { text, audioUrl: evt.data.audio_url },
        });
        options.onVRMSegment?.(evt.data);
      }

      if (evt.type === "vrm_complete") {
        options.onVRMComplete?.();
      }
    },
  });

  // 扩展 submit，自动注入 VRM 上下文
  const submitWithContext = useCallback((content: string) => {
    const worldState = useVRMWorldStore.getState();
    const vrmContext = {
      engagement: worldState.engagement,
      emotion: worldState.emotion?.current,
      gaze: worldState.gaze?.target,
    };

    director?.dispatch({ type: 'ENTER_THINKING' });

    stream.submit({
      messages: [{ type: "human", content }],
      vrm_context: vrmContext,
    });
  }, [stream, director]);

  return { ...stream, submitWithContext };
}
```

#### `StreamAdapter` 类
对话层与指挥层之间的桥梁，负责将 useStream 的数据转化为 VRMCommand。

```typescript
// vrm-director/StreamAdapter.ts

export class StreamAdapter {
  constructor(private director: VRMDirector) {}

  /**
   * 当 useStream.toolCalls 变化时调用
   * 从 perform_actions 工具中提取立即执行的命令
   */
  handleToolCalls(toolCalls: ToolCallWithResult[]) {
    for (const tc of toolCalls) {
      if (tc.call.name !== 'perform_actions') continue;
      if (tc.state !== 'pending') continue;

      const actions = tc.call.args.actions as ActionItem[];
      const commands = adaptToolActions(actions);
      commands.forEach(cmd => this.director.dispatch(cmd));
    }
  }

  /**
   * 当收到 VRM custom event 时调用
   */
  handleVRMSegment(segment: { marked_text: string; audio_url: string | null }) {
    const { text, commands } = parsePerformanceTags(segment.marked_text);

    // 旁路命令（情感/动作）→ 立即执行
    commands.forEach(cmd => this.director.dispatch(cmd));

    // 叙事命令 → 排队
    this.director.dispatch({
      type: 'NARRATE_SEGMENT',
      payload: { text, audioUrl: segment.audio_url },
    });
  }

  /**
   * 构建提交时的场景上下文快照
   */
  buildContext(): VRMContext {
    const state = useVRMWorldStore.getState();
    return {
      character: {
        engagement: state.engagement,
        emotion: state.emotion.current,
        posture: state.posture,
        gaze: state.gaze.target,
      },
      camera: state.camera,
    };
  }
}
```

## 三、Layer 2: 指挥层（Director Layer）

### 3.1 职责
- 接收 VRMCommand，按类型和优先级路由给子系统
- 管理角色交互状态机（XState）
- 编排叙事队列（音频 → 字幕 → 动作 → 表情 的顺序执行）
- 隔离业务逻辑与渲染逻辑

### 3.2 核心组件

#### `VRMDirector` 主指挥者

```typescript
// vrm-director/VRMDirector.ts

class VRMDirector {
  // 音频元素（Director 持有，通过 getter 暴露给 R3F LipSync）
  private audioElement: HTMLAudioElement;

  // 命令路由器
  private router: CommandRouter;

  // 交互状态机（XState）
  private engagementSM: EngagementStateMachine;

  // 子系统控制器
  private narrative: NarrativeController;
  private expression: ExpressionController;
  private motionBridge: MotionBridge;
  private camera: CameraController;
  private gaze: GazeController;
  private sfx: SFXController;
  private scene: SceneController;

  // 状态写入
  private store: typeof useVRMWorldStore;

  constructor(store: typeof useVRMWorldStore) {
    this.audioElement = new Audio();
    this.store = store;
    // ... 初始化各子系统
  }

  /** 暴露给 R3F 层用于 LipSync */
  getAudioElement(): HTMLAudioElement {
    return this.audioElement;
  }

  /** 统一命令入口 */
  dispatch(command: VRMCommand): void {
    this.router.route(command);
  }

  /** 立即打断，清空所有队列 */
  interrupt(): void {
    this.narrative.clear();
    this.audioElement.pause();
    this.engagementSM.send('INTERRUPT');
  }

  destroy() {
    this.audioElement.pause();
    this.audioElement.src = '';
  }
}
```

**为什么 AudioElement 由 Director 持有而不放在 Store 里：**
1. `HTMLAudioElement` 是可变 DOM 对象，不是纯数据，不应存在 Zustand Store
2. Zustand devtools 无法序列化 DOM 对象
3. Director 天然拥有音频播放的完整生命周期控制权

#### `CommandRouter` 命令路由器

按命令类型和优先级分发给不同子系统（详见 [命令系统设计](../命令系统设计.md)）。

#### 子系统控制器

每个控制器专注一个子系统，暴露简洁的接口：

| 控制器 | 职责 | 执行方式 |
|---|---|---|
| `NarrativeController` | 管理叙事队列（音频+字幕+动作的顺序播放） | FIFO 队列，异步等待 |
| `ExpressionController` | 管理情感层表情（happy/sad/...） | 立即执行，带渐变 |
| `MotionBridge` | 将动作指令写入 Store，R3F 层读取执行 | 写 Store |
| `CameraController` | 镜头切换和过渡 | 立即执行 |
| `GazeController` | 注视目标控制 | 立即执行 |
| `SceneController` | 背景/地面配置 | 写 Store |
| `SFXController` | 音效播放（可叠加） | 立即执行 |

## 四、Layer 3: 渲染层（Rendering Layer）

### 4.1 职责
- 从 Store 读取状态，驱动 Three.js/R3F 渲染
- 运行自治程序（眨眼、口型、LookAt）
- 将 R3F 层事件（如 motionFinished）回写到 Store

### 4.2 核心组件

#### `VRMWorldStore` — 全局状态

Store 只描述"世界当前的状态"，**不包含任何业务逻辑**。

```typescript
interface VRMWorldStore {
  // ===== Director 实例 =====
  director: VRMDirector | null;

  // ===== 多实体注册表 =====
  entities: Record<string, EntityState>;  // 所有场景实体

  // ===== 主角色交互状态 =====
  engagement: EngagementState; // idle_initial / thinking / narrating / ...

  // ===== 子系统状态（只读快照）=====
  emotion: { current: string; intensity: number };
  motion: { url: string | null; category: string | null };
  subtitle: { text: string; visible: boolean };
  camera: { mode: string; dist: string };
  gaze: { target: string };
  posture: string; // standing / sitting

  // ===== 配置（持久化）=====
  renderConfig: VRMRenderConfig;     // 光照/后处理
  sceneConfig: VRMSceneConfig;       // 背景/地面
  characterConfig: VRMCharacterConfig; // 位置/缩放/旋转
  cameraConfig: VRMCameraConfig;     // 相机参数

  // ===== 事件通道（R3F -> Director 的单向通知）=====
  _events: {
    motionFinished: number; // 时间戳，变化即代表事件触发
  };
  _fireMotionFinished: () => void;
}

// ===== 多实体状态结构 =====
interface EntityState {
  id: string;
  type: 'vrm' | 'prop' | 'static_gltf';
  url: string;
  isMain?: boolean;            // 是否是主 AI 角色
  position: [number, number, number];
  rotation: [number, number, number];
  scale: number;
  parentId?: string;           // 挂载到父实体（如角色手持道具）
  boneName?: string;           // 挂载点骨骼名
  visible: boolean;
  vrmState?: {                 // 仅 VRM 类型有
    emotion: string;
    motion: string;
  };
}
```

#### R3F 组件

```
Character.tsx (R3F 组件)
├── 读 Store.motion → 执行动作
├── motionFinished → 写 Store._events
├── useAutoBlink — 自治程序，不经过 Director
├── useLipSync — 自治程序，需 audioElement ref
└── useAutoLookAt — 读 Store.gaze.target
```

**自治程序说明：**
- `useAutoBlink`、`useLipSync` 在 `useFrame` 中每帧运行
- 它们只需要 VRM 实例和/或 audioElement 引用，**不经过 Director**
- 这是因为它们是 60fps 实时程序化驱动，提交到 Director 会增加不必要的延迟

### 4.3 R3F ↔ Director 通信

采用**混合架构**：

| 通信类型 | 方式 | 原因 |
|---|---|---|
| **Director → R3F（"什么该播"）** | 通过 Store（状态化，可观察） | 无时序要求的数据 |
| **Director → R3F（"怎么播"）** | 通过 ref + 回调 bridge（命令式） | 需要精确时序控制 |
| **R3F → Director（"播完了"）** | 写 Store._events 事件字段 | Director 通过 Zustand subscribe 监听 |

```
┌────────────── Director ──────────────┐
│  MotionBridge          AudioCtrl     │
│  → 写 Store.motion     → 管理Audio  │
└────┬──────────────────────┬──────────┘
     │ Store                │ ref
     ↓                      ↓
┌────────────── R3F ───────────────────┐
│  Character.tsx                       │
│  ├── 读 Store.motion → play(url)    │
│  ├── onFinished → 写 Store._events  │
│  └── useLipSync(audioElement ref)   │
└──────────────────────────────────────┘
```

#### motionFinished 事件的具体实现

```typescript
// VRMWorldStore 中的事件通道
_events: { motionFinished: 0 },
_fireMotionFinished: () => set(state => ({
  _events: { ...state._events, motionFinished: Date.now() }
})),

// Character.tsx (R3F层) — 触发事件
const fireMotionFinished = useVRMWorldStore(s => s._fireMotionFinished);
motionController.onFinished(() => { fireMotionFinished(); });

// NarrativeController (指挥层) — 监听事件
const unsubscribe = useVRMWorldStore.subscribe(
  (state) => state._events.motionFinished,
  (timestamp) => {
    if (this.waitingForMotion) {
      this.waitingForMotion = false;
      this.resolveMotionWait?.();
    }
  }
);
```

> **注意：** 此方案使用 Zustand 的 `subscribe` selector 形式，在 Class 内部使用，**不需要 React useEffect**。有一帧延迟，但对于动作完成这类事件是可接受的。

## 五、多实体渲染（Entity Factory）

### 5.1 设计理念

场景可能同时存在多个模型：主角色（VRM）、道具模型（GLTF）、静态装饰等。渲染层通过 **实体工厂模式** 从 `entities` 注册表动态渲染。

### 5.2 AIStage 中的实体工厂

```typescript
function AIStage() {
  const entities = useVRMWorldStore(s => s.entities);

  return (
    <>
      <SceneLighting />
      <Background />
      <Ground />

      {/* 实体工厂：根据类型渲染不同组件 */}
      {Object.values(entities).map((entity) => {
        if (!entity.visible) return null;
        switch (entity.type) {
          case 'vrm':
            return <Character key={entity.id} config={entity} />;
          case 'prop':
            return <Prop key={entity.id} config={entity} />;
          case 'static_gltf':
            return <StaticModel key={entity.id} config={entity} />;
          default:
            return null;
        }
      })}
    </>
  );
}
```

### 5.3 角色 vs 道具组件差异

| | `Character.tsx` (type=vrm) | `Prop.tsx` (type=prop) |
|---|---|---|
| 表情系统 | 完整三层（情感/口型/眨眼） | 无 |
| 骨骼动画 | 完整（idle/thinking/reply...） | 基础（单一循环动画） |
| LookAt | 程序化注视 | 无 |
| 状态机 | XState 交互状态 | 无 |
| 挂载 | 可作为父（被挂载） | 可作为子（挂到角色骨骼） |

### 5.4 挂载机制（Attachment）

当道具需要"手持"时，Director dispatch `ATTACH_OBJECT` 命令，修改 Store 中对应实体的 `parentId` 和 `boneName`。渲染层的 `Prop` 组件检测到 `parentId` 后，通过 `forwardRef` 找到父骨骼建立父子级关系。

## 六、资源管理

### 6.1 模型加载/卸载

- **加载驱动**：当 `entities[id].url` 变化时，对应 R3F 组件自动触发 React 重挂载
- **卸载自动化**：React 卸载组件时调用 `vrm.dispose()` 或遍历 GLTF scene 释放显存
- **缓存策略**：`useGLTF` (drei) 内置 URL 级别缓存，同一 URL 不重复加载

### 6.2 动作缓存池

`useMotionController` 内部维护 `Map<url, AnimationClip>` 缓存，设 LRU 上限（如 30 个），超出淘汰最早的 clip。

### 6.3 背景列表来源

- 背景列表由后端 `GET /api/v1/assets/backgrounds` 提供
- 应用初始化时拉取存入 `sceneConfig.availableBackgrounds`
- AI 可通过 `perform_actions` 切换

## 七、状态归属原则

**核心：AI 需感知 或 需持久化 -> 全局 Store；纯渲染细节 -> 组件本地**

| 状态类型 | 归属 | 原因 |
|---|---|---|
| 当前模型 URL / 动作 URL | **全局 Store (entities)** | AI 需通过 Context 感知；持久化恢复 |
| 背景 / 地面 / 镜头配置 | **全局 Store** | 持久化；AI 可通过 `perform_actions` 修改 |
| 情感 / 交互状态 | **全局 Store** | AI 需感知；Director 需驱动状态机 |
| 骨骼插值权重 / Blend 进度 | **组件本地 (Ref)** | 60fps 高频更新，AI 无需知道 |
| 模型加载进度 % | **组件本地 (State)** | 临时 UI 反馈 |
| Audio AnalyserNode 频谱 | **组件本地 (Ref)** | useLipSync 内部，纯渲染驱动 |

## 八、预览模式（VRMPreviewer）

针对模型预览、动作预览场景，提供**不依赖 Director/通信层的轻量级组件**：

```typescript
// components/vrm/VRMPreviewer.tsx

interface VRMPreviewerProps {
  modelUrl: string;
  motionUrl?: string;     // 可选：预览的动作
  loop?: boolean;         // 动作是否循环，默认 true
  enableBlink?: boolean;  // 是否启用自动眨眼，默认 true
  enableLookAt?: boolean; // 是否启用自动注视，默认 false
  width?: string;
  height?: string;
}

function VRMPreviewer(props: VRMPreviewerProps) {
  return (
    <Canvas style={{ width: props.width, height: props.height }}>
      <Suspense fallback={null}>
        <Character
          config={{
            id: 'preview', type: 'vrm', url: props.modelUrl,
            isMain: false, position: [0,0,0], rotation: [0,0,0],
            scale: 1, visible: true,
          }}
          overrideMotion={props.motionUrl ? { url: props.motionUrl, loop: props.loop } : undefined}
          enableBlink={props.enableBlink}
          enableLookAt={props.enableLookAt}
        />
        <Environment preset="city" />
        <OrbitControls />
      </Suspense>
    </Canvas>
  );
}
```

**设计要点：**
- **复用渲染层**：`Character.tsx` 里的对齐、眨眼逻辑在预览时直接可用
- **不依赖 Director**：通过 `overrideMotion` prop 直接传入动作
- **不依赖 Store**：不订阅 `VRMWorldStore`，完全自包含
- **场景简化**：使用 `OrbitControls` 替代 `GenshinControls`，更适合模型查看
