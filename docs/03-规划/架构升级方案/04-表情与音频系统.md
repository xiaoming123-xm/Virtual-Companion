# 表情与音频系统

---

## 一、VRM 表情三层系统

### 1.1 核心概念

VRM 的 `expressionManager` 是一个**混合器（Blender）**，多个表情值可以叠加：

```
expressionManager.setValue('happy', 0.8)   // 情感层
expressionManager.setValue('aa', 0.4)       // 口型层（说话时）
expressionManager.setValue('blink', 1.0)    // 眨眼层（眨眼时）
```

这三个**完全独立、同时存在**，是 VRM 表情系统的关键特性。

### 1.2 三个独立层

```
┌────────────────────────────────────────────────────────────┐
│  层1：情感层（Emotion Layer）                               │
│  驱动方：Director → ExpressionController                   │
│  内容：happy / sad / angry / surprised / neutral           │
│  时机：AI标签<emo v='happy'/>触发，持续到下次情感命令      │
│  特点：变化慢，有渐变过渡（blend duration 200ms）          │
├────────────────────────────────────────────────────────────┤
│  层2：口型层（LipSync Layer）                              │
│  驱动方：useLipSync（R3F内部，每帧运行）                   │
│  内容：aa（张嘴幅度，0~1）                                 │
│  时机：有TTS音频时自动运行，音频停止时归零                 │
│  特点：60fps 实时，完全程序化，不经过 Director             │
├────────────────────────────────────────────────────────────┤
│  层3：眨眼层（Blink Layer）                                │
│  驱动方：useAutoBlink（R3F内部，每帧运行）                 │
│  内容：blink（闭眼幅度，0~1）                              │
│  时机：随机间隔（2-6秒）                                   │
│  特点：完全程序化，不经过 Director                          │
└────────────────────────────────────────────────────────────┘
```

**关键结论：** 层2 和 层3 根本不需要经过 Director。它们是 R3F 层的「常驻程序」，只需要拿到 VRM 实例（和/或 audioElement）就能自运行。**Director 只管情感层。**

### 1.3 ExpressionController 接口

```typescript
interface ExpressionController {
  /**
   * 设置情感表情
   * @param name - 情感名 ('happy' | 'sad' | 'angry' | ...)
   * @param intensity - 强度 0~1，默认 1.0
   * @param blendMs - 渐变时间，默认 200ms
   */
  set(name: string, intensity?: number, blendMs?: number): void;

  /** 重置为 neutral */
  reset(): void;

  /**
   * 渐变过渡
   * @param from - 当前情感
   * @param to - 目标情感
   * @param duration - 过渡时长
   */
  blend(from: string, to: string, duration: number): Promise<void>;
}
```

### 1.4 情感表情列表

| 情感 | VRM BlendShape | 典型触发方式 |
|---|---|---|
| neutral | neutral | 默认 / `<emo v='neutral'/>` |
| happy | happy | `<emo v='happy'/>` |
| sad | sad | `<emo v='sad'/>` |
| angry | angry | `<emo v='angry'/>` |
| surprised | surprised | `<emo v='surprised'/>` |
| excited | happy (高强度) | `<emo v='excited' i='1.0'/>` |
| relaxed | relaxed | `<emo v='relaxed'/>` |
| confused | confused | THINKING 状态时可选 |
| embarrassed | embarrassed | `<emo v='embarrassed'/>` |

## 二、音频双通道设计

### 2.1 叙事音频（Narrative Audio）

```
特点：
- 只有"有声"和"无声"两种状态，不重叠
- 与字幕严格同步
- 与动作可以联动（音频播完 → 可以等待动作也播完）
- 管理者：NarrativeController
- 底层：HTMLAudioElement（Director 持有）
```

### 2.2 特效音频（SFX Audio）

```
特点：
- 可以叠加（多个 AudioBuffer 同时播放）
- 立即响应，不排队
- 管理者：SFXController（独立于叙事队列）
- 底层：Web Audio API AudioBufferSourceNode
```

## 三、NarrativeController 内部工作流

### 3.1 流程描述

```
push(segment) 被调用

if (队列空闲):
  取头部 segment 开始处理：

  Step 1: dispatch SET_EMOTION（旁路，立即，不等待）
  Step 2: dispatch PLAY_MOTION（指定 motionUrl 写入 Store，不等待）
  Step 3: 写 Store.subtitle.text（立即）
  Step 4: 播放 NarrativeAudio（await onended）
            ↓ 音频结束
  Step 5: if (waitForMotion) → await Store._events.motionFinished
            ↓ 动作也完成（或超时 10s）
  Step 6: 清除字幕
  Step 7: 取队列下一个 segment，回到 Step 1
            ↓ 队列清空
  Step 8: 自动 dispatch ENTER_IDLE
```

### 3.2 接口定义

```typescript
class NarrativeController {
  private queue: NarrativeSegment[] = [];
  private isProcessing = false;
  private waitingForMotion = false;
  private resolveMotionWait?: () => void;

  constructor(
    private director: VRMDirector,
    private audioElement: HTMLAudioElement,
    private store: typeof useVRMWorldStore,
  ) {
    // 监听 motionFinished 事件
    this.store.subscribe(
      (state) => state._events.motionFinished,
      () => {
        if (this.waitingForMotion) {
          this.waitingForMotion = false;
          this.resolveMotionWait?.();
        }
      }
    );
  }

  /** 添加叙事片段到队列 */
  push(segment: NarrativeSegment): void {
    this.queue.push(segment);
    if (!this.isProcessing) {
      this.processNext();
    }
  }

  /** 添加等待到队列 */
  pushWait(ms: number): void {
    this.queue.push({ type: 'wait', ms });
  }

  /** 清空队列并停止播放 */
  clear(): void {
    this.queue = [];
    this.isProcessing = false;
    this.audioElement.pause();
    this.audioElement.currentTime = 0;
    this.store.setState({ subtitle: { text: '', visible: false } });
  }

  /** 核心处理循环 */
  private async processNext(): Promise<void> {
    if (this.queue.length === 0) {
      this.isProcessing = false;
      // 叙事完成，通知状态机
      this.director.dispatch({ type: 'ENTER_IDLE' });
      return;
    }

    this.isProcessing = true;
    const segment = this.queue.shift()!;

    if (segment.type === 'wait') {
      await delay(segment.ms);
      return this.processNext();
    }

    // Step 1: 情感（旁路立即执行）
    if (segment.emotion) {
      this.director.dispatch({
        type: 'SET_EMOTION',
        emotion: segment.emotion,
        intensity: segment.emotionIntensity,
      });
    }

    // Step 2: 动作（写入 Store）
    if (segment.motionName) {
      this.director.dispatch({
        type: 'PLAY_MOTION',
        motion: segment.motionName,
        category: segment.motionCategory,
      });
    }

    // Step 3: 字幕
    this.store.setState({ subtitle: { text: segment.text, visible: true } });

    // Step 4: 播放音频
    if (segment.audioUrl) {
      await this.playAudio(segment.audioUrl);
    }

    // Step 5: 等待动作完成（可选）
    if (segment.waitForMotion) {
      await this.waitForMotionComplete(10000); // 10s 超时
    }

    // Step 6: 清除字幕
    this.store.setState({ subtitle: { text: '', visible: false } });

    // Step 7: 下一个
    return this.processNext();
  }

  private playAudio(url: string): Promise<void> {
    return new Promise((resolve) => {
      this.audioElement.src = url;
      this.audioElement.onended = () => resolve();
      this.audioElement.onerror = () => resolve(); // 容错
      this.audioElement.play().catch(() => resolve());
    });
  }

  private waitForMotionComplete(timeoutMs: number): Promise<void> {
    return new Promise((resolve) => {
      this.waitingForMotion = true;
      this.resolveMotionWait = resolve;
      setTimeout(() => {
        if (this.waitingForMotion) {
          this.waitingForMotion = false;
          resolve();
        }
      }, timeoutMs);
    });
  }
}
```

## 四、R3F 层自治程序

### 4.1 useLipSync（口型同步）

- **输入：** audioElement（来自 Director.getAudioElement()）
- **运行方式：** `useFrame` 每帧更新
- **工作原理：** 通过 Web Audio API 的 AnalyserNode 分析音频频谱，提取元音数据驱动 VRM 的 `aa` BlendShape
- **生命周期：** 有音频播放时自动运行，音频停止时自动归零

### 4.2 useAutoBlink（自动眨眼）

- **输入：** VRM 实例
- **运行方式：** `useFrame` 每帧更新
- **工作原理：** 随机间隔（2-6秒）触发一次快速的眨眼动作（blink BlendShape 从 0→1→0）
- **生命周期：** VRM 实例存在时持续运行

### 4.3 useAutoLookAt（自动注视）

- **输入：** VRM 实例 + Store.gaze.target
- **运行方式：** `useFrame` 每帧更新
- **工作原理：** 读取 Store 中的注视目标，程序化控制 VRM 的 LookAt 系统
- **独立于 Director：** 不需要命令，只读 Store 状态
