# 状态机设计

---

## 一、设计思路

### 为什么需要状态机

当前的动作状态（initial/idle/thinking/reply）是用 `useRef` + 手动管理的，脆弱且不可观察。新架构引入 XState 状态机，确保：

1. **状态转换是唯一合法的变更路径**，任何命令只有在当前状态允许时才能执行
2. **状态可观察**，便于调试和 DevTools 查看
3. **支持并行状态维度**（交互、情感、注视、姿态可以独立变化）

### 选择 XState 的原因
- 真正的状态机库，非手写 `switch/case`
- 支持并行状态（parallel states）
- 支持延迟事件（delayed transitions）
- 内置 DevTools
- TypeScript 友好

## 二、并行状态维度

角色可以**同时**处于多个维度的状态：
- 正在 "NARRATING" && 情感 "happy" && 注视 "user"
- 正在 "THINKING" && 姿态 "sitting"
- 这些维度独立变化，互不阻塞

```
┌──────────────── VRM 角色世界状态 ────────────────────────┐
│                                                          │
│  ① 交互状态（Engagement）    [互斥，同一时刻只有一个]    │
│     IDLE_INITIAL → 待机，播放初始动作（循环）            │
│     IDLE_BORED   → 无聊闲置（超时触发），单次动作        │
│     LISTENING    → 用户正在输入/说话                     │
│     THINKING     → 等待AI响应（思考动作，循环）          │
│     NARRATING    → AI说话中（回复动作+音频+字幕）        │
│     INTERRUPTED  → 被用户中断，短暂过渡                  │
│                                                          │
│  ② 情感状态（Emotion）       [互斥，同一时刻只有一个]    │
│     neutral / happy / sad / angry / surprised            │
│     excited / relaxed / confused / embarrassed           │
│     + intensity（强度，0~1）                             │
│     控制：AI的<emo/>标签 或 Director推断                 │
│                                                          │
│  ③ 姿态状态（Posture）       [互斥]                      │
│     STANDING（默认）/ SITTING / LYING                    │
│     影响哪些动作库可以被选择                             │
│                                                          │
│  ④ 注视状态（Gaze）          [互斥]                      │
│     AT_USER（默认，程序化LookAt）                        │
│     LOOKING_AROUND（闲置时随机看）                       │
│     AT_TARGET（看场景物体，需要targetId）                 │
│     FOCUSED（专注，固定正前方）                          │
│                                                          │
│  ⑤ 可见性（Visibility）                                  │
│     VISIBLE / HIDDEN / FADING_IN / FADING_OUT            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 三、交互状态机（Engagement）— 核心

### 3.1 状态转换图

```
  加载完成
     │
     ▼
IDLE_INITIAL ──[超时9-15s]──→ IDLE_BORED
     │  ▲                          │
     │  └──────[动作结束]──────────┘
     │
     │[用户输入开始]
     ▼
 LISTENING ──[用户取消]──→ IDLE_INITIAL
     │
     │[用户提交消息]
     ▼
 THINKING ──[收到AI回复]──→ NARRATING ──[播放完毕]──→ IDLE_INITIAL
     │                          │
     │[用户中断]                │[用户中断]
     ▼                          ▼
 INTERRUPTED ──────────────────────────→ IDLE_INITIAL
```

### 3.2 XState 实现

```typescript
import { createMachine } from 'xstate';

const engagementMachine = createMachine({
  id: 'engagement',
  initial: 'unloaded',

  context: {
    idleTimerDelay: 12000,     // 闲置计时器基础延迟
    idleTimerVariance: 3000,   // 计时器随机波动
  },

  states: {
    unloaded: {
      on: {
        LOAD_CHARACTER: 'loading',
      },
    },

    loading: {
      on: {
        LOAD_SUCCESS: 'idle_initial',
        LOAD_FAILURE: 'error',
      },
    },

    idle_initial: {
      entry: ['playInitialMotion', 'startIdleTimer'],
      exit: ['stopIdleTimer'],
      on: {
        IDLE_TIMEOUT: 'idle_bored',
        USER_INPUT_START: 'listening',
        USER_SUBMIT: 'thinking',
        LOAD_CHARACTER: 'loading',
      },
    },

    idle_bored: {
      entry: ['playBoredMotion'],
      on: {
        MOTION_FINISHED: 'idle_initial',
        USER_INPUT_START: 'listening',
        USER_SUBMIT: 'thinking',
      },
    },

    listening: {
      entry: ['setListeningState'],
      on: {
        USER_SUBMIT: 'thinking',
        USER_CANCEL: 'idle_initial',
      },
    },

    thinking: {
      entry: ['playThinkingMotion', 'clearNarrativeQueue'],
      on: {
        NARRATE_START: 'narrating',
        INTERRUPT: 'interrupted',
        THINKING_TIMEOUT: 'idle_initial',  // 超时保护
      },
    },

    narrating: {
      entry: ['startNarration'],
      on: {
        NARRATE_COMPLETE: 'idle_initial',
        INTERRUPT: 'interrupted',
      },
    },

    interrupted: {
      entry: ['stopNarration', 'playInterruptAnim'],
      after: {
        500: 'idle_initial',  // 500ms 过渡后回到 idle
      },
    },

    error: {
      entry: ['logError'],
      on: {
        RETRY: 'loading',
        RESET: 'unloaded',
      },
    },
  },
});
```

### 3.3 命令 → XState 事件映射

```
Director 收到命令               XState 事件           状态转换
────────────────────────────────────────────────────────────────
LOAD_CHARACTER                → LOAD_CHARACTER       → loading
(模型加载完毕)                → LOAD_SUCCESS         → idle_initial
ENTER_LISTENING               → USER_INPUT_START     → listening
ENTER_THINKING                → USER_SUBMIT          → thinking
NARRATE_SEGMENT (第一个)      → NARRATE_START        → narrating
(叙事队列清空)                → NARRATE_COMPLETE     → idle_initial
INTERRUPT / STOP_ALL          → INTERRUPT            → interrupted
(闲置计时器触发)              → IDLE_TIMEOUT         → idle_bored
(无聊动作播完)                → MOTION_FINISHED      → idle_initial
```

## 四、状态影响系统行为的对照表

| 交互状态 | 动作行为 | 表情行为 | 注视行为 | 可接受命令 |
|---|---|---|---|---|
| IDLE_INITIAL | 循环初始动作 | 维持当前情感 | 看用户（轻微drift） | 全部 |
| IDLE_BORED | 单次闲置动作 | 随机轻微情感 | 随机看周围 | 全部 |
| LISTENING | 保持当前动作 | 专注neutral | 看用户/跟随鼠标 | THINKING, STOP |
| THINKING | 循环思考动作 | confused可选 | 轻微随机 | NARRATE, INTERRUPT |
| NARRATING | 跟随TTS分段 | AI标签控制 | 看用户 | INTERRUPT（打断） |
| INTERRUPTED | 中断淡出 | 维持当前 | 看用户 | RESET→IDLE |

## 五、并行状态 XState 结构

```typescript
// 完整的并行世界模型
const vrmWorldMachine = createMachine({
  type: 'parallel',
  states: {
    engagement: engagementMachine,  // 上面定义的

    emotion: {
      initial: 'neutral',
      states: {
        neutral: {}, happy: {}, sad: {}, angry: {},
        surprised: {}, excited: {}, relaxed: {},
        confused: {}, embarrassed: {},
      },
      on: {
        SET_EMOTION: {
          actions: ['updateEmotion'],
        },
      },
    },

    gaze: {
      initial: 'at_user',
      states: {
        at_user: {}, looking_around: {},
        at_target: {}, focused: {},
      },
      on: {
        SET_GAZE: { actions: ['updateGaze'] },
      },
    },

    posture: {
      initial: 'standing',
      states: {
        standing: {}, sitting: {},
      },
      on: {
        SET_POSTURE: { actions: ['updatePosture'] },
      },
    },
  },
});
```

> **注意：** 情感、注视、姿态三个维度可以作为 XState 的 parallel states 实现，也可以作为独立的简单状态存储在 Store 中（因为它们的转换逻辑非常简单）。建议**只有 Engagement 使用 XState**，其他维度直接在 Store/Controller 中管理。
