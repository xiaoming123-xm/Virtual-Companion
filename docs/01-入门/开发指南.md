# 开发指南

本文档用于说明 ATRI Chat 的本地开发、调试与构建流程。若只想快速找到相关文档，请先查看 [文档中心](../README.md)。

## 🛠️ 前置要求

在开始之前，请确保你的系统已安装：
- **Python 3.12+** (推荐)
- **Node.js 18+**
- **Rust** 工具链 (用于 Tauri 构建)
- **FFmpeg** (用于音频编解码处理)
- [uv](https://github.com/astral-sh/uv) (强烈推荐，用于 Python 依赖管理)

---

## 🏃 快速启动 (开发模式)

### 1. 克隆并进入目录
```bash
git clone https://github.com/1sunxiaoshou/atri-chat.git
cd atri-chat
```

### 2. 后端服务 (FastAPI)
我们使用 `uv` 管理依赖，这会比 `pip` 快得多。
```bash
# 安装依赖并同步虚拟环境
uv sync

# 启动后端 (默认端口 9099)
uv run main.py
```

### 3. 前端界面 (React + Vite)
```bash
cd frontend
npm install
npm run dev
```
访问控制台输出的地址即可开始调试。

## 📦 打包发布 (Production Build)

我们提供了一个统一的发布脚本 `scripts/release.py`，它可以一键完成前端编译、后端打包、Tauri 客户端封装及 ZIP 发布包生成。

关于安装版与便携版的数据目录策略、运行模式划分、发布文件命名和版本迭代规则，请参考：
- [桌面构建与发布指南](./桌面构建与发布指南.md)

### 统一构建工具用法
```bash
# 执行完整流水线（交互模式，会提示输入版本号）
uv run python scripts/release.py

# 命令行指定版本打包完整版
uv run python scripts/release.py release --format all --app-version 1.0.1

# 发布前预检
uv run python scripts/release.py check

# 仅清理构建产物
uv run python scripts/release.py clean

# 同步并检查发布版本
uv run python scripts/bump_version.py 1.0.2
uv run python scripts/bump_version.py --check
```

### 构建产出说明
执行完成后，根目录的 `release_package/` 文件夹下会生成：
- `atri-chat-{version}-windows-x64-setup.exe`：安装版程序。
- `atri-chat-{version}-windows-x64-portable.zip`：绿色便携版压缩包。
- `atri-chat-{version}-sha256sums.txt`：发布文件校验清单。

## 🏗️ 项目架构

- **`core/`**: 系统核心逻辑，包括 ASR, TTS, VRM 调度。
- **`api/`**: 暴露给前端的 RESTFUL 接口。
- **`frontend/`**: 基于 React 19 的单页应用。
- **`frontend/src-tauri/`**: Rust 壳程序，负责窗口管理及后端进程守护。
- **`data/`**: (运行时生成) 所有的用户数据、模型、配置均存放在此，实现真正的 Portable。
