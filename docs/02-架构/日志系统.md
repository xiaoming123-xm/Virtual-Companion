# 日志系统

本文档说明日志文件划分、配置方式和使用建议。

## 核心原则

- **分离关注点** - 不同类型日志分文件存储
- **异步写入** - 避免阻塞请求
- **结构化格式** - 易于解析和查询

## 日志文件

| 文件       | 用途         | 格式                                    | 保留期 |
| ---------- | ------------ | --------------------------------------- | ------ |
| access.log | HTTP 请求    | 时间\|IP\|请求ID\|方法 路径\|状态\|耗时 | 7 天   |
| app.log    | 应用运行记录 | 时间\|级别\|模块:函数:行号 - 消息       | 7 天   |
| error.log  | 错误和堆栈   | 时间\|级别\|模块:函数:行号 - 消息+堆栈  | 30 天  |
| slow.log   | 慢请求(>1s)  | 时间\|请求ID\|方法 路径\|耗时           | 30 天  |



## 配置

```bash
LOG_LEVEL=INFO   # 可选，默认根据环境自动设置
```

### 日志级别

| 运行环境    | 级别    | 说明     |
| ----------- | ------- | -------- |
| development | DEBUG   | 详细日志 |
| production  | WARNING | 精简日志 |

### 慢查询阈值

在 `core/middleware/logging_middleware.py` 修改:

```python
SLOW_THRESHOLD_MS = 1000  # 默认 1 秒
```



## 常用命令

```bash
# 查看最近错误
tail -f logs/error.log

# 统计接口访问量
cat logs/access.log | awk '{print $7}' | sort | uniq -c | sort -rn

# 查看所有慢查询
cat logs/slow.log

# 追踪特定请求 (通过请求ID)
grep "a1b2c3d4" logs/*.log

# 统计状态码分布
cat logs/access.log | awk '{print $9}' | sort | uniq -c
```



## 最佳实践

### 日志级别使用

| 级别     | 使用场景       | 示例                 |
| -------- | -------------- | -------------------- |
| DEBUG    | 详细调试信息   | 变量值、函数调用     |
| INFO     | 关键业务操作   | 用户登录、创建订单   |
| WARNING  | 可恢复的异常   | 重试成功、降级处理   |
| ERROR    | 需要关注的错误 | API 失败、数据库错误 |
| CRITICAL | 系统级别故障   | 服务崩溃、数据丢失   |

### 应该记录

- 请求 ID（用于追踪）
- 关键业务操作
- 错误和异常（含堆栈）
- 性能指标（响应时间）

### 不应该记录

- 用户密码
- 完整 API 密钥
- 个人敏感信息
- 大量重复的调试信息
