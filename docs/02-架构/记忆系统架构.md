# 记忆系统架构

## 概述

本文档说明长期记忆系统的组织方式、工具接口和自动加载机制。

新的长期记忆系统采用 **文档形式** 维护记忆。记忆以 Markdown 文件形式存储，支持：
- 按角色隔离（不同角色的记忆完全独立）
- 三层记忆结构：状态层、事件层、概念层
- 相对路径支持，AI 可自主管理文件结构
- 短期记忆保持不变（使用 LangGraph Checkpointer）

## 设计原则

1. **最小化工具数量**：4 个核心工具，通过参数控制行为
2. **工具即文档**：工具描述包含完整使用说明
3. **灵活的文件管理**：支持相对路径，AI 可创建子目录和归档
4. **极简 System Prompt**：只说明记忆系统的存在，具体用法在工具描述中

## 文件结构

```
data/memory/{character_id}/
├── user_profile.md       # [状态层] 用户核心档案（热数据）
├── memory_stream.md      # [事件层] 时间线流水账（温数据）
├── topics.md             # [概念层] 深度话题摘要（冷数据）
└── archive/              # [归档] AI 可自主创建
    └── 2026_Q1.md
```

## 核心工具（4个）

### 1. memory_read(file, section=None, tail_lines=None)
**统一的读取工具**

支持三种读取模式：
- 完整读取：`memory_read(file="user_profile")`
- 章节读取：`memory_read(file="user_profile", section="Basic Info")`
- 尾部读取：`memory_read(file="memory_stream", tail_lines=50)`

### 2. memory_write(file, content, section=None, mode="append")
**统一的写入工具**

支持两种写入模式：
- 追加模式：`memory_write(file="memory_stream", content="...", mode="append")`
- 替换模式：`memory_write(file="user_profile", section="Basic Info", content="...", mode="replace")`

### 3. memory_search(query, file=None, limit=10)
**统一的搜索工具**

支持两种搜索范围：
- 全局搜索：`memory_search(query="创业项目")`
- 单文件搜索：`memory_search(query="创业", file="memory_stream")`

### 4. memory_list(file=None)
**列出文件或章节**

支持三种列出模式：
- 列出所有文件：`memory_list()`
- 列出文件章节：`memory_list(file="user_profile")`
- 列出话题：`memory_list(file="topics")`


## 自动加载机制

### 用户画像自动加载

系统在构建提示词时自动加载用户画像：

```python
# core/prompts/prompt_manager.py
def build_character_prompt(...):
    # ...
    # 组件 4: 用户画像（自动加载）
    user_profile = self._load_user_profile(character_id)
    if user_profile:
        prompt_parts.append(user_profile)
```

**智能加载**：
- 如果 `user_profile.md` 有实际内容 → 自动添加到 System Prompt
- 如果档案为空或只有模板 → 不加载（避免浪费 token）

**判断标准**：
- 包含 5 个以上 "(未知)" → 认为是空档案
- 否则 → 认为有实际内容

### 提示词组装顺序

```
1. 核心身份 (角色设定)
2. 渲染协议 (文本模式 / VRM 模式)
3. 记忆系统说明
4. 用户画像 (自动加载) ← 关键
5. 安全准则
6. 额外指令
```

### AI 的视角

每次对话开始时，AI 会自动看到：
1. 角色设定（名字、性格、背景）
2. 输出协议（如何回复）
3. 记忆系统说明（有哪些工具）
4. **用户画像**（用户的基本信息、偏好等）← 自动加载

AI 无需手动调用 `memory_read(file="user_profile")`，画像会自动出现在上下文中。

### 其他记忆文件

- `memory_stream` - 事件流水账：AI 需要时主动调用 `memory_read()` 或 `memory_search()`
- `topics` - 话题摘要：AI 需要时主动调用 `memory_read()`

只有 `user_profile` 自动加载，因为它是核心信息且相对较小。

## 三层记忆文件

### user_profile.md（状态层）
存储用户的核心信息，章节包括：
- `[Basic Info]` - 基本信息
- `[Personality]` - 性格特点
- `[Preferences]` - 偏好
- `[Social]` - 社交关系
- `[Current]` - 当前状态

### memory_stream.md（事件层）
按时间顺序记录事件，格式：
```
- [YYYY-MM-DD HH:MM] (Category) 事件描述
```

常用分类：Chat, Fact, Event, Project, Decision

### topics.md（概念层）
深度话题的结构化摘要，格式：
```markdown
## [Topic: 话题名称]
- **关键点 1**: 内容
- **关键点 2**: 内容
```

## 相对路径支持

AI 可以自主创建子目录和新文件：
```python
# 创建归档
memory_write(file="archive/2026_Q1", content="...", mode="append")

# 创建专题文件
memory_write(file="projects/startup", content="...", mode="append")

# 读取归档
memory_read(file="archive/2026_Q1")
```

## 使用示例

### 场景 1: 用户自我介绍
```python
# 1. 更新档案
memory_write(
    file="user_profile",
    section="Basic Info",
    content="- Name: 李明\n- Career: 设计师",
    mode="replace"
)

# 2. 记录事件
memory_write(
    file="memory_stream",
    content="- [2026-03-07 14:30] (Chat) 用户自我介绍：李明，设计师",
    mode="append"
)
```

### 场景 2: 回忆往事
```python
# 全局搜索
memory_search(query="创业项目")

# 或只搜索流水账
memory_search(query="创业", file="memory_stream")
```

### 场景 3: 整理话题
```python
# 整理成话题摘要
memory_write(
    file="topics",
    section="Topic: 创业项目",
    content="- **项目愿景**: AI 伴侣\n- **技术栈**: Python, FastAPI",
    mode="replace"
)
```

### 场景 4: 归档旧记录
```python
# AI 可以自主决定何时归档
memory_write(
    file="archive/2026_Q1",
    content="[归档内容]",
    mode="append"
)
```

## System Prompt

极简的 System Prompt（约 200 字）：

```markdown
# 记忆系统

你拥有长期记忆能力，可以记住用户信息和对话历史。

记忆以 Markdown 文件形式存储：
- user_profile.md - 用户档案
- memory_stream.md - 事件流水账
- topics.md - 话题摘要

你可以自由创建子目录和新文件来组织记忆。

使用原则：
1. 及时记录用户信息
2. 保持档案简洁
3. 结构化组织
4. 自主管理文件

工具的具体用法请参考工具描述。
```

## 优势总结

1. **工具数量少**：4 个工具 vs 原来的 8 个
2. **参数化控制**：通过参数选择行为，而不是创建多个工具
3. **灵活的文件管理**：支持相对路径，AI 可自主组织
4. **工具即文档**：工具描述包含完整使用说明
5. **极简 System Prompt**：只需 200 字说明记忆系统的存在
6. **人类可读**：Markdown 格式，易于查看和编辑
7. **易于备份**：文件系统，支持版本控制
