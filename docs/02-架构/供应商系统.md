# 供应商系统

## 概述

本文档说明模型供应商接入方式、能力识别规则和图标系统设计。

## 已集成供应商（10 个）

### 国际供应商
1. **OpenAI** - GPT 系列模型（gpt-4o, o1-preview, o3-mini）
2. **Anthropic** - Claude 系列（claude-4-opus, claude-3.5-sonnet）
3. **Google** - Gemini 多模态模型（gemini-2.5-pro, gemini-2.0-flash）
4. **Mistral AI** - 欧洲开源模型（mistral-large, codestral）
5. **DeepSeek** - 高性能开源模型（deepseek-chat, deepseek-reasoner）
6. **Cohere** - 企业级 AI 平台（command-r-plus）
7. **Groq** - 超高速推理引擎（llama-3.3-70b, mixtral-8x7b）
8. **xAI** - Grok 系列模型（grok-2, grok-2-vision）

### 国内供应商
9. **Tongyi (通义千问)** - 阿里百炼平台（qwen-plus, qwen-max, qwq-plus）
10. **Local (Ollama)** - 本地模型（llama3, qwen, mistral 等）

## 核心特性

### 1. 自动能力识别

使用 LangChain 的 `model.profile` 功能自动识别模型能力，无需手动维护映射表。

支持的能力类型：
- **VISION** - 图像输入
- **DOCUMENT** - 文档理解
- **AUDIO** - 音频输入/输出
- **VIDEO** - 视频输入
- **TOOL_USE** - 工具调用
- **REASONING** - 深度推理

### 2. 统一图标系统

使用 `@lobehub/icons` 替代上传图片方案：

```typescript
import { ProviderIcon } from '@/src/components/ProviderIcon';

<ProviderIcon providerId="openai" size={24} />
```

优势：
- 统一视觉风格，自动深浅色主题
- 零存储开销，更快加载
- 支持 30+ 供应商图标
- 自动回退到首字母缩写

### 3. 通用参数

所有供应商支持：
- **temperature** - 温度 (0-2)
- **top_p** - 核采样 (0-1)
- **max_tokens** - 最大输出 token 数

### 4. 供应商特定参数

| 供应商    | 参数名           | 说明                    |
| --------- | ---------------- | ----------------------- |
| OpenAI    | reasoning_effort | 推理强度：低/中/高      |
| Anthropic | thinking         | 思考配置                |
| Google    | thinking_config  | 思考预算和输出控制      |
| Tongyi    | enable_thinking  | 启用深度思考（boolean） |

## 技术实现

### 能力识别流程

```python
def get_model_info(self, model_id: str, provider_config: ProviderConfig):
    """优先使用 LangChain profile，降级到名称推断"""
    model_instance = self.create_text_model(model_id, provider_config)
    
    if hasattr(model_instance, 'profile') and model_instance.profile:
        # 从 profile 提取能力
        profile = model_instance.profile
        capabilities = []
        if profile.get('image_inputs'):
            capabilities.extend([VISION, DOCUMENT])
        if profile.get('tool_calling'):
            capabilities.append(TOOL_USE)
        # ...
    else:
        # 降级到名称推断
        capabilities = self._infer_from_name(model_id)
```

### 图标组件实现

```typescript
const PROVIDER_ICON_MAP: Record<string, React.ComponentType<any>> = {
  openai: OpenAI,
  anthropic: Anthropic,
  google: Google,
  qwen: Qwen,
  deepseek: DeepSeek,
  // ... 30+ 供应商
};

export function ProviderIcon({ providerId, size = 24 }: Props) {
  const IconComponent = PROVIDER_ICON_MAP[providerId.toLowerCase()];
  
  return IconComponent ? (
    <IconComponent size={size} />
  ) : (
    <div>{providerId.slice(0, 2).toUpperCase()}</div>
  );
}
```
## 扩展指南

### 添加新供应商

1. 创建 Provider 类继承 `BaseProvider`
2. 实现必需方法：`create_text_model()`, `list_models()`
3. 注册到 `ModelFactory`
4. 在 `PROVIDER_ICON_MAP` 中添加图标映射

```python
# core/models/providers/newprovider.py
class NewProvider(BaseProvider):
    def create_text_model(self, model_id: str, config: ProviderConfig):
        return ChatNewProvider(
            model=model_id,
            api_key=config.api_key,
        )
```

```typescript
// frontend/src/components/ProviderIcon.tsx
import { NewProvider } from '@lobehub/icons';

const PROVIDER_ICON_MAP = {
  newprovider: NewProvider,
};
```

### 添加新图标

如果 `@lobehub/icons` 不支持，组件会自动显示首字母回退。可以提交 PR 到 [lobehub/lobe-icons](https://github.com/lobehub/lobe-icons) 添加新图标。

## 优势总结

1. **自动化** - LangChain profile 自动识别能力
2. **统一性** - 统一的图标系统和视觉风格
3. **可扩展** - 新增供应商只需实现接口
4. **零维护** - 无需手动维护能力映射和图标文件
5. **高性能** - SVG 图标更轻量，零存储开销
